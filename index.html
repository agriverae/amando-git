¿Qué es un repositorio de Git?

Un repositorio de Git es simplemente una base de datos que contiene toda la información necesaria para retener y manejar las revisiones e historia de un proyecto.

El proceso de agregar nuevos cambios al repositorio se lograr a través de varias etapas

¿Cómo crear un nuevo repositorio de Git?

Para poder empezar a usar los comandos y funcionalidades especiales de git. Debemos ejecutar el siguiente comando 

              
              git init

Conceptos Generales Parte 1

En el corazón de Git se encuentra la tienda de objetos (Object Store)

Contiene la información original de todos tus archivos, mensajes de log, información del autor, fechas y cualquier otra información necesaria para rearmar cualquier versión o rama de un proyecto

Tiene 4 tipos de objetos: Blobs, arboles, commits y tags

Blob

  • Binary
  • Large
  • OBject

Es un termino para referirse a un archivo o variable que puede contener cualquier tipo de data y su estructura interior es ignorada por el programa.

No contiene ninguna información respecto del archivo. En git cada versión de un archivo es representado como un Blob.

Trees (Arboles)

Representa un nivel de información del directorio.

Guarda los identificadores de los blobs, nombre de rutas y un poco de metada respecto de los archivos dentro de un directorio.

También puede hacer referencia a otros arboles, creando una jerarquía de arboles.

Commits

Guarda metada de cualquier cambio introducido al repositorio, incluyendo autor, mensaje, etc.

Cada commit apunta a un Arbol que captura en una "foto" el estado del repositorio en ese momento dado.

El primer commit inicial no tiene padre.

En general, los commits tienen al menos un padre.

Tags

Asigna un nombre legible para los humanos a un objecto especifico, usualmente un commit.

Index

Recuerda y retiene los cambios que se pasaron a "Stage". Manteniendolos a salvo hasta que uno les haga commit

El indice permite una transición gradual, usualmente guiado por el desarrollador. Desde un estado complejo del repositorio a otro estado presuntamente mejor

Conceptos Generales Parte 3

Nombres Basados en contenido

Cada objeto(blobs, arboles, commits y tags) tiene un nombre unico que se produce al aplicar una encriptación a los contenidos del mismo. Dandonos un valor del tipo SHA1.

SHA1 son valores de 160 bits que usualmente se representan con un número hexadecimal de 40 digitos.

El mismo contenido en diferentes directorios hasta en otras maquinas dan como resultado el mismo SHA1.

Conceptos Generales Parte 4

Git rastrea el contenido

Cuando git guarda un archivo en la tienda de objetos lo hace basado en el valor computado(SHA1) y no el nombre del archivo.

Si dos archivos diferentes tienen el mismo tipo de contenido, independientemente de donde esten ubicados. Git guarda una única copia del archivo como un blob dentro de la tienda de objetos indexado por un valor del tipo SHA1

¿Qué sucede cuando un archivo cambia?

Git computa un nuevo identificador del tipo SHA1 y determina que es un nuevo tipo de "Blob" en la tienda de objetos. El "blob" original se mantiene intacto.

Git guarda cada version de cada archivo. Dado que git usa un valor computado basado en los contenidos del archivo como nombre del archivo debe operar en cada copia completa del archivo

Conceptos Generales Parte 5

Paquete de archivos

Git usa un sistema muy eficiente de guardado. Para crear un paquete de archivos, git primero localiza los archivos cuyo contenido sea muy similar y guarda el contenido completo de uno de ellos

Luego calcula las diferencias o deltas y las guarda

De esta manera construye los otros archivos usando el archivo que se uso como base y luego le aplica las diferencias o deltas

Ejemplos de la tienda de objetos

Blob: Es el último objeto del fondo(abajo); No hace referencia a nada y solo es referenciado por arboles. En las imagenes se distingue por un rectangulo.

Arboles: Apuntan a blobs y posiblemente a otros arboles también. Cualquier Arbol puede ser referenciado por uno o más commits. Cada arbol es representado por un triangulo

Commit: Es representado por un circulo. Apunta a un arbol en particular que fue introducido al repositorio por ese mismo commit

Tag: Es representado por un paralelogramo.

Branch(Rama): no es un Objeto de git fundamental, pero juega un rol crucial a la hora de nombrar un commit. Cada branch(rama) es representado por un rectangulo ovalado
